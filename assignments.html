Javascript != Java

Javascript is a programming language developed in the late 1990s by Netscape. Microsoft said no.

It's similar to most other programming languages, variables, loops, logic, functions. The difference is the browser engine.

Modern browsers let you practice javascript inside the browser in real time. How is this different from a language like C++?
Javascript is still compiled, but it is compiled in the browser by the browser's javascript engine.

This means we can test our javascript in the browser before saving it to our .js files!

Logic pt. 1: If Statements

So we can create these variables, but that's useless unless we add a little logic into it. Let's start with 'if', 'else' statements.

  • 'If' statements let you run some code based on whether or not some condition passes. Here's the syntax: 
    if( SOME CONDITION TO CHECK ) { 
        CODE TO RUN IF CONDITION PASSES 
    }
    Here's an example: 
    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    }
  • What if we only want something to happen if the 'if' condition fails? 'Else' statements! 
    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    } else {
        var itIsOver = 0;
    }
  • We've secretly just introduced a NEW data type, BOOLEANS! Booleans are a data type that can only ever have 1 of 2 different values, true or false. Booleans are used in 'if' statements and loops. Here's a few examples of Booleans: 
    true
    false
    In addition to those two, comparison expressions can be evaluated as a boolean. Try putting these in your console and see what you get: 
    10 > 5
    10 > 15
    var myNameString = "My name is Josh Soileau";
    myNameString === "My name is Josh Soileau"
    var myNameString = 123456;
    myNameString === "My name is Josh Soileau"

Arrays

Let's learn about yet another data type in javascript: arrays. Arrays are one of the most useful things in javascript, they let you organize things in groups.

  • Arrays are defined by square brackets []. When the compiler sees those square brackets, it knows to expect an array of items inside the brackets. 
    [ ARRAY ITEMS GO IN HERE ]
  • The items inside the brackets are all separated by commas, to signify the dividers to the compiler. 
    [ arrayItem, anotherArrayItem, yetAnotherArrayItem ]
    In the above case we have an array of size 3.
  • Create an array of strings. Make it have at least 4 items in it, and make each item be your favorite topping on a pizza. Save that array to a variable. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
  • Okay, so we now have an array with values in it. How do we get those values later? We can access arrays using INDEXES
  • What are indexes? They are slots in the array! But the secret is that the first slot of the array is index "0" (like all things comp sci) (insert explanation of starting at 0 and not 1).
    Think of it this way:
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ]

       Index 0    Index 1        Index 2         Index 3        Index 4
  • So the way we access indexes of the array are by using indexes in the array. Here's the syntax: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[ NUMBER OF INDEX I WANT TO ACCESS ]
  • To get the first index of the array we would use the 0th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[0];
  • To get the third index of the array we would use the 2nd index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[2];
  • How do we get the last item in the array? In this case we know it's the 4th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[4];
    But what if we didn't know how long our array was? What if somebody inserts a 6th item in the array without us knowing?
    You can learn how long an array is by calling ".length" on it like this; 
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ].length
    Which is the same thing as saying: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings.length
    Notice how it returns the number "5". But our last index is index #4! So to get the last element in the array, we could do this: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];

    pizzaToppings[ pizzaToppings.length - 1 ];
  • It's important to note that arrays don't have to hold just 1 type of data. You can have an array of anything! 
    [ "Cheese", 42, 999 ]
    [ "Rocky", true, false, "Bullwinkle" ]
    You can even have an array of arrays! 
    [ [12, 32], ["Cherry", "Vanilla", false] ]

Logic pt. 2: For Loops!

Sometimes you want to run the same bit of code over and over again. There are a couple of ways to do that in javascript, we are going to start with For loops!

  • For loops are similar to if statements. You have some parenthesis and some curly braces. The code inside the curly braces is the code that runs during the loop. Here's the syntax: 
    for( BEFORE CODE ; WHILE CONDITION ; AFTER CODE ) {
        CODE TO BE RUN DURING LOOP 
    }

    BEFORE CODE: this is set at the beginning of the loop
    It is usually something like var i=0 or something like that

    WHILE CONDITION: the condition to keep running the loop

    AFTER CODE: this runs after every iteration of the loop
  • Here's a simple example of for loops: 
    var i;
    var concatNumbersVar = "";
    for( i=0; i<6; i++) {
        concatNumbersVar = concatNumbersVar + i;
    }
    At the top, we create two variables, one isn't defined yet (i), the other is an empty string. The loop starts at i = 0, and loops through as long as i < 6, but increasing i by 1 every time the loop finishes.
    The code inside the loop takes our empty string, and appends a number on the back of it every time the loop happens (remember what happens when you add a string and a number together?)

All about functions!

Sometimes in programming we have to repeat ourselves multiple times. When it comes to that kind of thing, we have a rule in programming. It's called D.R.Y., DON'T REPEAT YOURSELVES. Luckily, in programming, we have a concept called functions that let us perform repetetive behaviors in a simple way.

  • Let's take a common use case. We want print out to a greeting to the user. If you did that without a function you would do something like this: 
    var userName = "Josh Soileau";
console.log('Hello, ' + userName);
console.log('It's nice to see you today.');
console.log('Looks like you've got a lot on your plate today.')
console.log('Better get to work!');
  • But if you had different userNames, you'd have to do all 5 lines over and over again for every user. Or, we could just use a function! Here's the function syntax: 
    var myFunctionName = function( PARAMETER TO GIVE THE FUNCTION ) {
  CODE INSIDE THE FUNCTION
};
    So then, whenever you wanted to run "CODE INSIDE THE FUNCTION", you would just call the function like this: 
    myFunctionName( SOME PARAMETER I WANT TO PASS );
    I know that can be confusing, let's see an example.
    Here's the code we had before, but this time in function form! 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
    IMPORTANT NOTE: Just writing that code out WILL NOT RUN that code.
    All we did in that code was tell the computer "If anyone ever calls the function 'greetTheUser', do this code right here".
    To actuall run the code, we have to "call" it, like this: 
    greetTheUser( "Josh Soileau" );
  • So what is the thing inside of our call to greetTheUser()?
    That is our "parameter".
    Functions work a bit like machines, you give them SOMETHING, and they do things with it!
    In this case, we are "giving" our greetTheUser function a string called "Josh Soileau".
    Let's look at our function again: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
    As you see, our function expects to be passed something (as you can see because there is a variable inside the parenthesis of the function declaration).
    We are saying that the greetTheUser function expects to be called and passed an argument, and we are going to call that argument 'userName'.
    The function doesn't know WHAT it is being passed, just that it is being passed SOMETHING.
    Exercise: Try calling the function with a different data type, like a number, array, or object. 
    greetTheUser( 12.34 );
greetTheUser( "HAL" );
    As you can see, it treats it the same way! It saves whatever it is being passed into this 'userName' variable, and tries to use it as well as it can.
  • Functions can also return values that we can save for later. See this example: 
    var timesTwo = function(number) {
    return number * 2;
};

var thisShouldEqualFourteen = timesTwo( 7 );
    The above function expects to receive a number, does some math, and gives back a new number.
    We can save the result in a variable. Super useful!

All about scope!

Lets take a look at a concept called "scoping".

  • Here's our previous function: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};
  • Inside this function we make a variable called "userName". What happens if we try and access that variable somewhere else, like this: 
    var greetTheUser = function( userName ) {
  console.log('Hello, ' + userName);
  console.log('It's nice to see you today.');
  console.log('Looks like you've got a lot on your plate today.')
  console.log('Better get to work!');
};

console.log( userName );
    What do we get there and why?
    In javascript, variables (and functions) are only available in the context they are defined in.
  • Think of scoping like each bit of code is a small piece of a neighborhood.
    Each "function" is a fenced in property. Anything on the inside, stays in the inside.
    In our example above, the "userName" variable is created inside our fenced in yard.
    But then we try and use that variable OUTSIDE of the fence, and it doesn't work. That is because OUTSIDE of the fence, we don't know what "userName" is, it is undefined!
  • The same thing will happen here as well: 
    var prepareForUser = function( userName ) {
  var greetingString = "Hello, " + userName;
  var userAge = 54;
  var userFavoriteColor = "Beige";
};

// ALWAYS REMEMBER TO CALL YOUR FUNCTION!
// Otherwise, the code inside it won't run! 
prepareForUser( "Josh Soileau" );

console.log( greetingString );
console.log( "Your age is " + userAge );
console.log( "Your favorite color is " + userFavoriteColor );
  • So how do we fix that code above??
    Remember how our code is kind of like a neighborhood with fences in it?
    Well, if we create a variable OUTSIDE of ALL of the fences, then that variable is available everywhere, even INSIDE the fences.
    That is called a variable being "global". Let's look at an example: 
    var wildNeighborhoodDog = "Fido";
var dogIsFarAway = 1;

var callDog = function( dogName ) {

  if( dogIsFarAway ) {
    console.log( "Here, " + dogName + ", here boy!" );
  } else {
    console.log( "Good boy, " + dogName );
  }

};

callDog( wildNeighborhoodDog );
    Because our "wildNeighborhoodDog" and "dogIsFarAway" variables are GLOBAL, we can use them inside our "callDog" function.

    Let's try and fix our code from before. We can solve our problem by making the variables global, like this: 
    var greetingString;
var userAge;
var userFavoriteColor;

var prepareForUser = function( userName ) {
  greetingString = "Hello, " + userName;
  userAge = 54;
  userFavoriteColor = "Beige";
};

// ALWAYS REMEMBER TO CALL YOUR FUNCTION!
// Otherwise, the code inside it won't run! 
prepareForUser( "Josh Soileau" );

console.log( greetingString );
console.log( "Your age is " + userAge );
console.log( "Your favorite color is " + userFavoriteColor );
    All we had to do is DECLARE our variables globally. We didn't even need to assign them any value.
    Just the act of declaring them outside the function meant that we could access them outside the function later on.
  • Now we know that we can make variables GLOBAL and have access to them everywhere. So why don't we do this for everything??
    Take a look at this code below, tell me what the output is: 
    var numberOfBalloons = 99;
var balloonsColor = "Red";

var popOneBalloon = function( balloonsColor ) {

  numberOfBalloons = numberOfBalloons - 1;
  var balloonsColor = "Blue";

};

console.log( numberOfBalloons + " " +  balloonsColor + " balloons go by." );
    What is the output of that code and why? (move the console.log around to learn more)
  • This type of thing is why it's a bad idea to have too many global variables. By saying that YOUR variable is global, you are effectively preventing other coders from using that variable name in different scopes. If you can, try and do without global variables.

Object basics

This is where you get your feet wet with objects. Objects are everywhere in JS. They are an additional datatype, like Number and Array. In fact, they are quite similar to arrays in that they are multiple, multidimensional data stores and can contain any type of data. You can have an objet of objects, and object of arrays, etc.

The fundamental difference between objects and arrays is their construction as key/value pairs. Where arrays are simple collections with a numerical key, objects take a specified keyword. Objects can be referenced either by square bracket notation or by dot syntax. More about that below.

We won't get too deep into objects in this class, just enough to get started working with the DOM. But understanding how to use objets is key to understanding more advanced JavaScript and learning how to use JS frameworks like jQuery, Angular, Node, Backbone, Underscore, Moment, Meteor, React... The list goes on (and on).

  • Basic Notation. Objects are defined using curly braces. Inside the curly braces, your object will contain one or many key/value pairs, separated by a comma. Each key and value is separated by a colon. The key may be written without quotes, with single-quotes or with double-quotes. 
    {
  key: 'value',
  "iAmAKey": "I am a value",
  'singleQuotes': "double quotes"
};
  • Accessing Values. To access the value of an object, simply refer to its key either via dot notation or by accessing its key in square brackets. 
    var dog = {
  breed: 'Great Dane',
  color: 'harlequin',
  food: 'small children'
};

// get the value using dot notation
dog.food;

// get the value using bracket notation
dog['food'];

// ^^^ note that we used food as a string in the last line.
  • Properties and Methods. In regular JS, we have variables and functions. In objects, we generally refer to properties and methods in the same way. While this isn't as strict as other programming languages, you're still going to run into the terminology. Properties describe the object. Methods do something. 
    var dog = {
  // properties

  breed: 'Great Dane',
  color: 'harlequin',
  food: 'small children',
  
  // methods

  bark: function() {
    alert('WOOF!');
  },
  
  getColor: function() {
    return this.color;
    // this refers to the context in which the funciton is called
    // this can get you into trouble
    // if you're really curious, look up JavaScript's call() funciton
  }
};

dog.bark();

dog.getColor();
  • Native Objects. JavaScript comes packages with a number of native objects like Math and Date. Back in the day, many designers worked with the Image object to preload mouseover images before you could really do that with CSS. Native objects ship with the browser (hence native) and are always there. 
    var someDate = new Date();
someDate.getFullYear(); // ZOMG!

// the Math object is really quite useful
console.log('The Life of ' + Math.PI);

// no, really
Math.round(Math.PI);

// by the way, this stuff is case sensitive, so watch out!
  • The document and window Objects. Here is where things get a little more fun, especially for those of you who are bored of working exclusively in the console. Go ahead and type the following into the browser console: 
    window.
    Notice the console is trying to autocomplete for you. Each autocomplete possibility is a member of the window object. With so many API's within the HTML spec, you have access to a TON of possibilities. Play with window.history and see what's in there.

    The same goes for the document object. We have access to all kinds of browser behaviors just through the native document object. This is how we'll manipulate the DOM later on in the class. For now, let's move on ... what is the DOM, anyway?
     

The Document Object Model

When you write an HTML document and load it in a browser, the browser interprets your HTML and builds a web page. The elements you write, the attributes in your elements and the text inside your elements are all nested together neatly in your HTML. The browser translates this structure into the page as a hierarchy of nodes and relationships. This is the DOM. It is simply a representation of the constructed web page itself, mapped to a hierarchical tree. Think of it as a map. The HTML you write is parsed by the browser and becomes the DOM. The DOM is the foundation of the page you see.

  • A Simple Page. Consider the following HTML. 
    <html>
  <head>
    <title>The World Cup is over</title>
    <link rel="icon" type="image/x-icon" href="/images/favicon.ico">
  </head>
  <body>
    <div class="container">
      <h1 class="header">The World Cup is, like, over</h1>
      <p style="color: #933;">Now we have to watch baseball. Ugh.</p>
      <p class="footnote">Just kidding! Baseball is totes exciting cuz they score lots.</p>
    </div>
  </body>
</html>
    Once the browser reads the markup and constructs the page, the DOM will look something like this. 
    html
  |-head
  | |-title
  | | |-"The World Cup is over"
  | |-link
  | | |-(rel="icon")
  | | |-(type="image/x-icon")
  | | |-(href="/images/favicon.ico")
  |-body
    |-div
    |-(class="container")
    |-h1
    | |-(class="header")
    | |-"The World Cup is, like, over"
    |-p
    | |-(style="color: #933;")
    | |-"Now we have to watch baseball. Ugh."
    |-p
      |-(class="footnote")
      |-"Just kidding! Baseball is totes exciting cuz they score lots."
    That is the DOM. You have element nodes, attribute nodes and text nodes. Technically, you could also have comment nodes, but let's stick with these three for now.
  • In the console. Open up a console window. When you look at you document structure (Elements tab in Chrome Inspector, HTML tab in Firefox), it looks like the HTML you would have written. But this series of collapsible HTML is a visual representation of the DOM.
     
    This is the interpreted DOM. As we'll see in the next section, we can use JavaScript to manipulate the DOM. The result of those manipulations will be different than the original HTML document. But the browser console will always reflect the current state of the DOM.
     

The DOM API

Okay, great. We have the DOM. So what? The browser downloads your HTML, styles are applied, images and fonts loaded. Without a line of JavaScript, the DOM exists and a polished web page gets painted to my browser. What good is JavaScript?

  • getElementById(). Change is good. The DOM API (an API is just a way for one program (e.g. ours) to talk to another (e.g. the browser)) lets us manipulate the page. Let's say we wanted to change the h1 element on a page. Before we can change the element, we have to select it. The easiest way is to select a node by its ID, since an ID can only appear once on a page.
     
    Open this codepen: http://codepen.io/mccambridge/pen/fHKmn. In the JavaScript window on the right, enter the following code. The result will automatically re-run. 
    var theHeader = document.getElementById('the-title');
alert(theHeader);
    getElementById() is a method on the document object. Notice the contents of the alert? It's an object. Every DOM element is an object. We'll learn what to do with that in a minute.
  • Accessing a Text Node Now that we have access to an element and remembering that text is a node in itself, how can we dive one level down into the text node? 
    var theHeader = document.getElementById('the-title');
alert(theHeader.innerHTML);
    innerHTML is a property of our HTML node object. It is simply a record of the HTML inside of this node. In the current case, the only HTML left is simple text. Let's try changing it. 
    var theHeader = document.getElementById('the-title');
theHeader.innerHTML = 'Well, why don\'t you write your own header.';
  • Other Ways To Access a Node The browser gives us a few other methods to grab hold of our HTML elements. Let's start with document.getElementsByTagName(). Notice it's 'elements' (plural). This method returns an array. Try the following in the CodePen. 
    var theHeader = document.getElementsByTagName('h1');
alert(theHeader.length);
    What happens if you add another h1 element? What if you want to manipulate the h1 elements? How would you accomplish that (remember looping around arrays from previous classes)?
     
    The API also provides document.getElementsByClassName() which also returns an array but takes — you guessed it! — a class name as its parameter.
     
    More recently, browser developers have included the document.querySelector() method, which takes a CSS selector, just like jQuery. Try this one in the CodePen: 
    var theHeader = document.querySelector('#the-title');
theHeader.innerHTML = 'This works with id, class and tag name. It selects the first instance of a match.'
    The API also provides document.querySelectorAll() which returns multiples. MDN does a good job documenting the API here.
  • Getting And Setting Attributes Aside from text, attributes are the other important side of HTML elements. Text populates our pages with content. Attributes give our elements options. Without attributes, img tags would be sourceless. Anchor tags would be href-less. What if we want to dive into selectors? Let's start with a fresh CodePen: http://codepen.io/mccambridge/pen/IxGDq. 
    var theKitteh = document.querySelector('#awsum-kitteh-pic');
alert(theKitteh.getAttribute('src'));
    Using the node's getAttribute() method, we can check to see what the img src attribute is set to. What if we want to change it up? 
    var theKitteh = document.querySelector('#awsum-kitteh-pic');
theKitteh.setAttribute('src', 'http://placekitten.com/900/200');
    setAttribute() takes two parameters. The first selects which parameter on the element is getting reset. The second tells the browser what to set it to. Now, you can change any attribute on any element. Meow!
