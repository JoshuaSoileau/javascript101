Javascript != Java

Javascript is a programming language developed in the late 1990s by Netscape. Microsoft said no.

It's similar to most other programming languages, variables, loops, logic, functions. The difference is the browser engine.

Modern browsers let you practice javascript inside the browser in real time. How is this different from a language like C++?
Javascript is still compiled, but it is compiled in the browser by the browser's javascript engine.

This means we can test our javascript in the browser before saving it to our .js files!

Logic pt. 1: If Statements

So we can create these variables, but that's useless unless we add a little logic into it. Let's start with 'if', 'else' statements.

  • 'If' statements let you run some code based on whether or not some condition passes. Here's the syntax: 
    if( SOME CONDITION TO CHECK ) { 
        CODE TO RUN IF CONDITION PASSES 
    }
    Here's an example: 
    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    }
  • What if we only want something to happen if the 'if' condition fails? 'Else' statements! 
    var oldLadyIsSinging = 1;

    if( oldLadyIsSinging ) {
        var itIsOver = 1;
    } else {
        var itIsOver = 0;
    }
  • We've secretly just introduced a NEW data type, BOOLEANS! Booleans are a data type that can only ever have 1 of 2 different values, true or false. Booleans are used in 'if' statements and loops. Here's a few examples of Booleans: 
    true
    false
    In addition to those two, comparison expressions can be evaluated as a boolean. Try putting these in your console and see what you get: 
    10 > 5
    10 > 15
    var myNameString = "My name is Josh Soileau";
    myNameString === "My name is Josh Soileau"
    var myNameString = 123456;
    myNameString === "My name is Josh Soileau"
    (insert explanation of === vs == )
  • All of these examples of booleans can be used inside an if statement as the condition to check!

Arrays

Let's learn about yet another data type in javascript: arrays. Arrays are one of the most useful things in javascript, they let you organize things in groups.

  • Arrays are defined by square brackets []. When the compiler sees those square brackets, it knows to expect an array of items inside the brackets. 
    [ ARRAY ITEMS GO IN HERE ]
  • The items inside the brackets are all separated by commas, to signify the dividers to the compiler. 
    [ arrayItem, anotherArrayItem, yetAnotherArrayItem ]
    In the above case we have an array of size 3.
  • Create an array of strings. Make it have at least 4 items in it, and make each item be your favorite topping on a pizza. Save that array to a variable. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
  • Okay, so we now have an array with values in it. How do we get those values later? We can access arrays using INDEXES
  • What are indexes? They are slots in the array! But the secret is that the first slot of the array is index "0" (like all things comp sci) (insert explanation of starting at 0 and not 1).
    Think of it this way:
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ]

       Index 0    Index 1        Index 2         Index 3        Index 4
  • So the way we access indexes of the array are by using indexes in the array. Here's the syntax: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[ NUMBER OF INDEX I WANT TO ACCESS ]
  • To get the first index of the array we would use the 0th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[0];
  • To get the third index of the array we would use the 2nd index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[2];
  • How do we get the last item in the array? In this case we know it's the 4th index. 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings[4];
    But what if we didn't know how long our array was? What if somebody inserts a 6th item in the array without us knowing?
    You can learn how long an array is by calling ".length" on it like this; 
    [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ].length
    Which is the same thing as saying: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];
    pizzaToppings.length
    Notice how it returns the number "5". But our last index is index #4! So to get the last element in the array, we could do this: 
    var pizzaTopings = [ "Cheese", "Pepperoni", "Green Peppers", "Red Peppers", "Mushrooms" ];

    pizzaToppings[ pizzaToppings.length - 1 ];
  • It's important to note that arrays don't have to hold just 1 type of data. You can have an array of anything! 
    [ "Cheese", 42, 999 ]
    [ "Rocky", true, false, "Bullwinkle" ]
    You can even have an array of arrays! 
    [ [12, 32], ["Cherry", "Vanilla", false] ]

Logic pt. 2: For Loops!

Sometimes you want to run the same bit of code over and over again. There are a couple of ways to do that in javascript, we are going to start with For loops!

  • For loops are similar to if statements. You have some parenthesis and some curly braces. The code inside the curly braces is the code that runs during the loop. Here's the syntax: 
    for( BEFORE CODE ; WHILE CONDITION ; AFTER CODE ) {
        CODE TO BE RUN DURING LOOP 
    }

    BEFORE CODE: this is set at the beginning of the loop
    It is usually something like var i=0 or something like that

    WHILE CONDITION: the condition to keep running the loop

    AFTER CODE: this runs after every iteration of the loop
  • Here's a simple example of for loops: 
    var i;
    var concatNumbersVar = "";
    for( i=0; i<6; i++) {
        concatNumbersVar = concatNumbersVar + i;
    }
    At the top, we create two variables, one isn't defined yet (i), the other is an empty string. The loop starts at i = 0, and loops through as long as i < 6, but increasing i by 1 every time the loop finishes.
    The code inside the loop takes our empty string, and appends a number on the back of it every time the loop happens (remember what happens when you add a string and a number together?)
